Binomial Negative Binomial Poisson Hypergeometric

Quick Decision Guide — Which Distribution?

Before diving into each distribution, use this flowchart mentally to identify which one applies.

Key Question Binomial Neg. Binomial Poisson Hypergeometric
Fixed n trials? ✅ Yes, fixed n ❌ Trials vary ❌ No trials ✅ Yes, fixed n
Replacement / independence? ✅ With replacement (or large pop.) ✅ With replacement N/A — events in time/space ❌ WITHOUT replacement
Counting what? # successes in n trials # trials until r-th success # events in interval # successes from finite pop.
Population size Infinite / large Infinite / large Infinite (events in time) Finite N, known K successes
💡
Golden Rule: If the problem says "drawn without replacement" from a finite population and you know the exact counts — use Hypergeometric. If "with replacement" or independent trials — use Binomial (or Negative Binomial if counting trials to a fixed # of successes).

Binomial Distribution

✅ When to Use

  • Fixed number of trials: n
  • Each trial is a success or failure
  • Probability of success p is constant every trial
  • Trials are independent
  • You are counting # of successes

⚙️ How to Apply

  1. Identify n (# trials) and p (success probability)
  2. Identify x (# successes you care about)
  3. Check: is p constant? Are trials independent?
  4. Plug into PMF below
  5. For ranges: sum multiple PMF values or use complement

Core Formulas

PMF — Probability Mass Function
P(X = x) = C(n, x) · px · (1 - p)n - x

Where: x = 0, 1, 2, ..., n
C(n, x) = n! / [x! · (n-x)!] ← "n choose x"
p = probability of success on one trial
Expected Value (Mean)
E[X] = n · p
Variance
Var(X) = n · p · (1 - p)
Standard Deviation
SD(X) = √(n · p · (1 - p))
Support
x ∈ {0, 1, 2, …, n}
⚠️
Common Mistake: Forgetting the binomial coefficient C(n,x). It counts the number of arrangements of x successes in n trials. Without it, you're only computing one specific arrangement.

Worked Examples

EASY Fair coin — probability of exactly 3 heads in 5 flips ▼

Setup: Flip a fair coin 5 times. Find P(X = 3).

1
Identify parameters: n = 5, p = 0.5, x = 3. ✅ Fixed trials, independent, binary outcome.
2
Compute the binomial coefficient: C(5,3) = 5! / (3! · 2!) = (5 · 4) / (2 · 1) = 10
3
P(X = 3) = C(5,3) · (0.5)3 · (0.5)2 = 10 · 0.125 · 0.25 = 10 · 0.03125
P(X = 3) = 0.3125 → 31.25%

Intuition: E[X] = 5(0.5) = 2.5, so 3 heads is just above average — makes sense it's the most common value.

MEDIUM Quality control — at least 2 defective items in a batch of 8 ▼

Setup: A factory produces items with a 15% defect rate. A batch of 8 is inspected. Find P(X ≥ 2).

1
n = 8, p = 0.15. Use the complement rule: P(X ≥ 2) = 1 − P(X = 0) − P(X = 1)
2
P(X = 0) = C(8,0) · (0.15)0 · (0.85)8 = 1 · 1 · 0.2725 = 0.2725
3
P(X = 1) = C(8,1) · (0.15)1 · (0.85)7 = 8 · 0.15 · 0.3206 = 0.3847
4
P(X ≥ 2) = 1 − 0.2725 − 0.3847 = 1 − 0.6572
P(X ≥ 2) ≈ 0.3428 → 34.28%
HARD Drug trial — expected value, variance, and a conditional probability ▼

Setup: A drug is 70% effective. 12 patients receive it. (a) Find E[X] and Var(X). (b) Given at least 1 patient is cured, what is the probability exactly 9 are cured?

1
n = 12, p = 0.70
E[X] = 12 · 0.70 = 8.4
Var(X) = 12 · 0.70 · 0.30 = 2.52
2
For part (b), use the conditional formula: P(X = 9 | X ≥ 1) = P(X = 9 and X ≥ 1) / P(X ≥ 1) = P(X = 9) / P(X ≥ 1)
3
P(X = 9) = C(12,9) · (0.7)9 · (0.3)3 = 220 · 0.040354 · 0.027 = 0.2397
4
P(X = 0) = (0.3)12 ≈ 0.000000531 ≈ 0
P(X ≥ 1) = 1 − P(X = 0) ≈ 1.0000
P(X = 9 | X ≥ 1) ≈ 0.2397 / 1.0000 ≈ 0.2397

Since P(X=0) is essentially 0, the conditional barely changes the answer — but the technique matters for cases where P(X=0) is non-trivial.

Negative Binomial Distribution

💡
Core Concept: You flip the question. Instead of "given n trials, how many successes?" you ask: "how many trials until the r-th success?" or "how many failures before r successes?"

✅ When to Use

  • Waiting for the r-th success (r is fixed)
  • Counting total trials OR # failures until r-th success
  • Trials are independent with constant p
  • The r-th success MUST occur on the last trial

⚙️ How to Apply

  1. Identify r (# successes desired) and p
  2. Decide: is x = total trials OR x = # failures?
  3. Apply the correct version of the PMF
  4. Note: the last trial is always a success

Two Common Parameterizations

Version A — X = Total Number of Trials to get r successes
P(X = x) = C(x - 1, r - 1) · pr · (1 - p)x - r
x = r, r+1, r+2, ... (x must be at least r)
Logic: Among the first (x-1) trials, exactly (r-1) were successes — the x-th trial is the r-th success.
Version B — Y = Number of Failures before the r-th success (Y = X - r)
P(Y = y) = C(y + r - 1, y) · pr · (1 - p)y
y = 0, 1, 2, ... (Note: C(y+r-1, y) = C(y+r-1, r-1))
E[X] — Total trials
E[X] = r / p
Var(X) — Total trials
Var(X) = r(1-p) / p²
E[Y] — # failures
E[Y] = r(1-p) / p
Var(Y) — # failures
Var(Y) = r(1-p) / p²
⚠️
Watch Out: Variance is the same for both X and Y, but the means differ by r. Also: when r = 1, you get the Geometric distribution — a special case of Negative Binomial!

Worked Examples

EASY Basketball — probability 5th free throw is the 3rd make (p = 0.8) ▼

Setup: A player makes free throws with probability 0.8. What is P(3rd make occurs on the 5th attempt)?

1
Use Version A. r = 3, p = 0.8, x = 5 (total trials). The 5th shot must be a make, and among the first 4 shots, exactly 2 were makes.
2
P(X = 5) = C(5-1, 3-1) · (0.8)3 · (0.2)5-3 = C(4,2) · (0.8)3 · (0.2)2
3
= 6 · 0.512 · 0.04 = 6 · 0.02048
P(X = 5) = 0.12288 ≈ 12.29%
MEDIUM Sales calls — expected calls until 4th sale, p = 0.3 ▼

Setup: A salesperson closes a deal with probability 0.3 per call. Find (a) the expected # of calls until the 4th sale, and (b) the variance.

1
r = 4, p = 0.3. Use Version A formulas.
2
E[X] = r / p = 4 / 0.3 = 13.33 calls
3
Var(X) = r(1-p) / p² = 4 · 0.7 / 0.09 = 2.8 / 0.09 = 31.11
E[X] ≈ 13.33 calls  |  Var(X) ≈ 31.11

Intuition: Each sale takes on average 1/p = 3.33 calls, so 4 sales ≈ 13.33 calls. Makes sense!

HARD Quality inspection — P(more than 5 failures before 3rd pass), p = 0.6 ▼

Setup: Items pass quality check with p = 0.6. Find P(Y > 5) where Y = # failures before the 3rd pass.

1
Use Version B. r = 3, p = 0.6. P(Y > 5) = 1 − P(Y ≤ 5) = 1 − [P(Y=0) + P(Y=1) + … + P(Y=5)]
2
General formula: P(Y = y) = C(y+2, y) · (0.6)3 · (0.4)y = C(y+2, 2) · 0.216 · (0.4)y
3
P(Y=0) = C(2,0)·0.216·1 = 1·0.216 = 0.21600
P(Y=1) = C(3,1)·0.216·0.4 = 3·0.0864 = 0.25920
P(Y=2) = C(4,2)·0.216·0.16 = 6·0.03456 = 0.20736
P(Y=3) = C(5,3)·0.216·0.064 = 10·0.013824 = 0.13824
P(Y=4) = C(6,4)·0.216·0.0256 = 15·0.0055296 = 0.082944
P(Y=5) = C(7,5)·0.216·0.01024 = 21·0.0022118 = 0.046450
4
P(Y ≤ 5) = 0.21600 + 0.25920 + 0.20736 + 0.13824 + 0.08294 + 0.04645 = 0.95019
P(Y > 5) = 1 − 0.95019 ≈ 0.0498 → ~5%

Poisson Distribution

✅ When to Use

  • Counting events in a fixed interval of time or space
  • Events occur independently
  • Events occur at a constant average rate λ
  • Two events cannot occur at the exact same moment
  • Also: large n, small p Binomial → approximate with Poisson (λ = np)

⚙️ How to Apply

  1. Identify the rate λ for the given interval
  2. If interval changes: scale λ proportionally
  3. Plug into PMF below
  4. For cumulative: sum or use complement

Core Formulas

PMF — Probability Mass Function
P(X = x) = (e−λ · λx) / x!

Where: x = 0, 1, 2, ... (no upper limit!)
λ = average rate of events per interval (must be > 0)
e ≈ 2.71828
Expected Value (Mean)
E[X] = λ
Variance
Var(X) = λ
Standard Deviation
SD(X) = √λ
Support
x ∈ {0, 1, 2, 3, …}
💡
Key Insight: In Poisson, mean = variance = λ. This is unique and a hallmark of the distribution. If you see a problem where they tell you both mean and variance and they're equal — it's almost certainly Poisson.

Scaling Lambda Across Intervals

Rate Scaling Rule
If rate is λ per T units, then for t units: λ_new = λ · (t / T)
Example: 3 arrivals/hour → for 2 hours: λ = 3 · 2 = 6
Example: 12 calls/hour → for 20 min (=1/3 hr): λ = 12 · (1/3) = 4

Worked Examples

EASY Emails — P(exactly 3 emails in an hour), λ = 5 ▼

Setup: You receive an average of 5 emails per hour. Find P(X = 3).

1
λ = 5, x = 3. The events (emails) are independent and occur at a constant average rate.
2
P(X = 3) = (e−5 · 53) / 3! = (0.006738 · 125) / 6
3
= 0.84224 / 6 = 0.14037
P(X = 3) ≈ 0.1404 → 14.04%
MEDIUM Car accidents — P(at most 2 in a 30-min period), λ = 2/hr ▼

Setup: Accidents occur at a rate of 2 per hour. Find P(X ≤ 2) in a 30-minute window.

1
Scale λ: 30 min = 0.5 hours. λ_new = 2 · 0.5 = 1
2
P(X ≤ 2) = P(X=0) + P(X=1) + P(X=2)
3
P(X=0) = e−1 · 10 / 0! = 0.3679
P(X=1) = e−1 · 11 / 1! = 0.3679
P(X=2) = e−1 · 12 / 2! = 0.1839
P(X ≤ 2) = 0.3679 + 0.3679 + 0.1839 = 0.9197 → 91.97%
HARD Call center — P(wait more than 2 minutes between calls), λ = 3/min (Poisson process insight) ▼

Setup: Calls arrive at rate λ = 3 per minute. What is P(no calls in next 2 minutes)? Also find E[X] and Var(X) over 4 minutes.

1
For "no calls in 2 minutes": scale λ to the 2-minute window. λ_2 = 3 · 2 = 6
2
P(X = 0 in 2 min) = e−6 · 60 / 0! = e−6 = 0.00248 → only 0.25% chance!
3
Over 4 minutes: λ_4 = 3 · 4 = 12
E[X] = λ_4 = 12 calls
Var(X) = λ_4 = 12
P(no calls in 2 min) ≈ 0.00248  |  E[X] = 12, Var(X) = 12 over 4 min

Hypergeometric Distribution

💡
Key Distinction from Binomial: The population is finite and you're sampling without replacement. Each draw changes the probability for the next draw. You MUST know the total population N and the number of "success" items K.

✅ When to Use

  • Finite population of size N
  • K "successes" in the population (known)
  • Sampling n items WITHOUT replacement
  • Counting X = # successes in the sample
  • Keywords: "committee," "batch," "deck of cards," "chosen from group"

⚙️ How to Apply

  1. Identify N (population), K (# successes in pop.), n (sample size)
  2. Check range: max(0, n+K-N) ≤ x ≤ min(n, K)
  3. Compute three separate combinations, then divide
  4. Numerator: (ways to choose successes) × (ways to choose failures)

Core Formulas

PMF — Probability Mass Function
P(X = x) = [C(K, x) · C(N-K, n-x)] / C(N, n)

N = total population size
K = number of success items in population
n = number of items drawn (sample size)
x = number of successes in the sample
Valid range: max(0, n+K-N) ≤ x ≤ min(n, K)
Expected Value (Mean)
E[X] = n · (K / N)
Variance
Var(X) = n · (K/N) · (1 - K/N) · [(N-n)/(N-1)]
Finite Population Correction
(N-n) / (N-1) ← always < 1
Comparison to Binomial
Var_Hyper = Var_Binom · [(N-n)/(N-1)]
⚠️
Finite Population Correction Factor (FPC): The variance is always smaller than the Binomial variance because without replacement, knowing what you've already drawn gives you information about what's left. As N → ∞, FPC → 1 and Hypergeometric approaches Binomial.

Worked Examples

EASY Deck of cards — P(exactly 2 aces in a 5-card hand) ▼

Setup: From a standard 52-card deck (4 aces), deal 5 cards. Find P(X = 2 aces).

1
N = 52, K = 4 (aces), n = 5 (cards dealt), x = 2. Without replacement → Hypergeometric.
2
P(X = 2) = [C(4,2) · C(48,3)] / C(52,5)
3
C(4,2) = 6
C(48,3) = 48!/(3!·45!) = (48·47·46)/(6) = 17,296
C(52,5) = 52!/(5!·47!) = (52·51·50·49·48)/120 = 2,598,960
4
P(X=2) = (6 · 17,296) / 2,598,960 = 103,776 / 2,598,960
P(X = 2) ≈ 0.03993 → ~3.99%
MEDIUM Committee — E[X] and Var(X) for women selected ▼

Setup: A group of 20 people has 8 women and 12 men. A committee of 5 is chosen at random. Find E[X] and Var(X) where X = # women on committee.

1
N = 20, K = 8 (women), n = 5, x = women selected.
2
E[X] = n · (K/N) = 5 · (8/20) = 5 · 0.4 = 2.0 women
3
Var(X) = n · (K/N) · (1 - K/N) · (N-n)/(N-1)
= 5 · 0.4 · 0.6 · (15/19)
= 5 · 0.4 · 0.6 · 0.7895
= 1.2 · 0.7895
E[X] = 2.0  |  Var(X) ≈ 0.9474

Note: Binomial Var would be 5(0.4)(0.6) = 1.2, which is larger. The FPC = 15/19 ≈ 0.789 reduces variance because sampling without replacement limits how extreme the outcomes can be.

HARD Lot inspection — P(X ≥ 2 defectives) from batch of 20 ▼

Setup: A lot has 20 items, 4 of which are defective. An inspector randomly selects 6 items without replacement. Find P(X ≥ 2 defectives).

1
N = 20, K = 4, n = 6. Use complement: P(X ≥ 2) = 1 − P(X=0) − P(X=1)
2
P(X=0) = [C(4,0) · C(16,6)] / C(20,6) = [1 · 8008] / 38760 = 0.2066
3
P(X=1) = [C(4,1) · C(16,5)] / C(20,6) = [4 · 4368] / 38760 = 17472/38760 = 0.4508
4
C(20,6) = 20!/(6!·14!) = 38,760
C(16,6) = 8,008  |  C(16,5) = 4,368
5
P(X ≥ 2) = 1 − 0.2066 − 0.4508 = 1 − 0.6574
P(X ≥ 2) ≈ 0.3426 → ~34.26%

Side-by-Side Master Comparison

Property Binomial Neg. Binomial Poisson Hypergeometric
Parameters n, p r, p λ N, K, n
X represents # successes in n trials # trials (or failures) until r-th success # events in interval # successes in sample of n
PMF C(n,x)·p^x·(1-p)^(n-x) C(x-1,r-1)·p^r·(1-p)^(x-r) e^(-λ)·λ^x / x! [C(K,x)·C(N-K,n-x)] / C(N,n)
E[X] np r/p λ n(K/N)
Var(X) np(1-p) r(1-p)/p² λ n·(K/N)·(1-K/N)·(N-n)/(N-1)
Replacement? Yes / independent Yes / independent N/A NO — without replacement
Fixed n? Yes No — n varies No — n not defined Yes
Special case Hypergeometric → Binomial as N→∞ r=1 → Geometric Binomial → Poisson when n large, p small Binomial when N very large

Flashcards

Click a card to reveal the answer.

What are the 4 conditions for a Binomial distribution?
↩ flip
1. Fixed n trials
2. Binary outcomes
3. Constant p
4. Independent trials
What does the binomial coefficient C(n,x) represent intuitively?
↩ flip
The number of ways to arrange x successes among n trials — it accounts for all possible orderings of the same outcome.
How does Negative Binomial differ from Binomial in what's "fixed" vs "variable"?
↩ flip
Binomial: fixed n, variable X (successes).
Neg. Binomial: fixed r (successes), variable X (trials needed).
What is the relationship between Poisson mean and variance?
↩ flip
E[X] = Var(X) = λ. Mean equals variance — this is unique to the Poisson distribution.
When can you approximate Binomial with Poisson?
↩ flip
When n is large (n ≥ 20) and p is small (p ≤ 0.05). Set λ = np.
What is the Finite Population Correction Factor in Hypergeometric and why is it ≤ 1?
↩ flip
(N-n)/(N-1). It's ≤ 1 because without replacement, successive draws are negatively correlated — this reduces variability compared to Binomial.
If r = 1 in the Negative Binomial, what distribution do you get?
↩ flip
Geometric distribution — counting trials until the first success.
Why does the last trial in Negative Binomial ALWAYS have to be a success?
↩ flip
By definition: we STOP when the r-th success occurs. So whatever trial we stop on must be a success — otherwise we'd have stopped earlier or later.
What is the valid range of x in the Hypergeometric distribution?
↩ flip
max(0, n+K-N) ≤ x ≤ min(n, K)
Lower bound: might be forced to pick some successes. Upper bound: limited by sample or population success count.
How do you scale λ in a Poisson problem when the time interval changes?
↩ flip
Scale proportionally: λ_new = λ_original × (new interval / original interval). Always convert to the same units first.
